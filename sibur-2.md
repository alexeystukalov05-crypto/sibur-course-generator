# ИИ-тьютор для оценки успеваемости сотрудников Сибура

**Отклик на запрос i.moscow/b2bshowcase/sibur — запрос №2**

---

## A — Проблема клиента

Сибур обучает тысячи сотрудников — но не знает, кто реально усвоил материал.

**Текущая боль:**

- Тесты показывают результат, но не причину ошибок
- Нет персональной обратной связи для каждого сотрудника
- Методисты не успевают анализировать успеваемость вручную
- Слабые места сотрудников остаются незамеченными до инцидента

> Нужен AI, который ведёт диалог, понимает где пробел, и даёт конкретную рекомендацию.

---

## B — Наше решение

AI-тьютор на базе LLM, который общается с сотрудником после курса и строит персональный дашборд успеваемости.

**Как это работает:**

1. Сотрудник проходит курс в LMS
2. AI-тьютор начинает диалог: задаёт вопросы по материалу
3. Анализирует ответы и выявляет пробелы в знаниях
4. Строит персональный дашборд: что усвоено, что нет
5. Даёт конкретные рекомендации по слабым местам
6. Методист видит сводный дашборд по всем сотрудникам

**Стек:**

- Claude API / Groq — диалог и анализ ответов
- Python / FastAPI — бэкенд
- React — дашборд успеваемости
- REST API — интеграция с LMS Сибура

---

## C — Демо и доказательства

**Что показываем:**

Прототип AI-тьютора — сотрудник отвечает на вопросы по курсу безопасности, тьютор анализирует ответы и строит мини-дашборд с оценкой по каждому разделу.

**Пример диалога:**

Тьютор: "Какие действия нужно предпринять при обнаружении утечки?"

Сотрудник отвечает → AI оценивает полноту → фиксирует пробел если ответ неполный

**Пример дашборда:**

- СИЗ: ✅ усвоено
- Химические вещества: ⚠️ частично
- Аварийные ситуации: ❌ требует повторения

---

## Почему это работает

- **Персонализация:** каждый сотрудник получает индивидуальную обратную связь
- **Скорость:** анализ успеваемости в реальном времени, без ручной работы методиста
- **Глубина:** AI видит не только правильно/неправильно, но и понимает причину ошибки
- **Масштаб:** один тьютор работает с тысячами сотрудников одновременно
